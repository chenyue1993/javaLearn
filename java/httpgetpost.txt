1.HTTP协议的详细内容
	1.1 什么是HTTP协议？
		- 超文本传输协议
		- 浏览器和服务器之间的一种通讯协议
		- 该协议有W3C负者制定的，其本质就是数据传输格式提前制定好。浏览器和服务器必须安装这种数据格式进行接收和发送。
	1.2 我们使用协议的版本号 HTTP1.1
	1.3 HTTP协议包括几部分？
		- 请求协议：从Browser发送到Server的时候才有的数据传输格式
		- 响应协议：从Server发送到Browser的时候采用的数据传输格式
	1.4 请求协议？
		请求协议包括四部分：
			请求行：- 请求方式 URI 协议版本号
			消息报头：
			空白行：- 专门用来分离消息报头和请求体
			请求体：
	1.5 响应协议？
			状态行：- 协议版本号 状态码 状态描述信息
			响应报头
			空白行：- 分离响应报头和响应体的
			响应体
			重点掌握状态码：        - 200  - 404  - 500
2.GET和POST请求的区别
	- 什么情况下浏览器发送的请求是GET请求？什么情况下浏览器发送的请求是POST请求？
		只有单使用表单form，并且将form标签的method属性设置为method="post"
		才是POST请求方式，其余剩下所有的请求方式都是基于GET方式的。
		
	- GET请求和POST请求有什么区别？
		- GET  在请求行上提交数据     格式：URI?name=value&name=value... 这种提交方式最终提交数据会显示到浏览器地址栏上
		- POST 在请求体上提交数据     格式：name=value&name=value...	         这种提交方式最终提交数据不会显示到浏览器地址栏上
		- POST 请求在请求体重提交数据，所以POST请求提交的数据没有长度限制【可以提交大数据】
		- GET  请求在请求行上提交数据，所以GET请求提交的数据有长度限制
		- GET请求只能提交字符串数据，POST请求可以提交任何类型的数据，包括视频。。。所以文件上传必须使用POST请求
		- GET请求最终的结果会被浏览器缓存收纳，而POST请求最终的结果不会被就浏览器缓存
				
	- GET请求和POST请求应当如何选择使用？
		- 必须使用POST请求的情况
			- 有敏感数据
			- 传输数据不是普通字符串字符串
			- 传输数据非常多
			- 这个请求是为了修改服务器端资源
		- GET请求大多是从服务器端读取资源，这个读取的资源短时间内不会发生变化，所以GET请求最终的结果浏览器缓存起来
		- POST请求大多是修改服务器端资源，而每次修改结果都是不同的，最终结果没有必要被浏览器缓存
		- 浏览器将资源缓存后，缓存的资源是和某个特定的路径绑定在一起的，只有浏览器再发送相同的请求路径，这时候就会去缓存中获取资源，
		   不在访问服务器，以这种方式降低服务器的压力，提高用户体验。但是有时候，我们不希望缓存，希望每次都访问服务器，可以在请求路径
		 后面添加时间戳，例如：http://ip:port/oa/login?timestamp=12312312312312
		 js怎么回去毫秒：new Date().getTime();
